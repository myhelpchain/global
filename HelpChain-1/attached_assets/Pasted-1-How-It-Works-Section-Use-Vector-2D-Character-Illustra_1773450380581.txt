1. “How It Works” Section – Use Vector 2D Character Illustrations

The images currently used in the “How It Works” section are good, but they should be replaced with vector-style 2D character illustrations instead of generic images.

Important:
These should specifically be clean vector illustrations, not photographs.

The goal is to create the kind of visuals commonly used in modern web applications and SaaS landing pages.

Each step in the “How It Works” section should include a vector 2D illustration that visually represents the action being described.

Examples of illustrations that should be used:

A person posting a task

Someone browsing available tasks

A helper completing a task

A user receiving help or confirming task completion

Requirements for the illustrations:

Must be vector-style 2D images

Must match a clean modern UI aesthetic

Must fit properly inside the existing containers/divs

Must visually align with the overall HelpChain design style

Preferred approach:

Search online for high-quality vector 2D illustrations, or

Generate them if suitable illustrations cannot be found.

The illustrations should feel consistent with each other and visually cohesive across the entire section.

2. “Browse Popular Tasks” Section – Add Placeholder Images

In the Browse Popular Tasks section, every task card should include a placeholder image.

These should be realistic images that represent the type of task, similar to the images already used in the Find Tasks page.

The idea is to maintain visual consistency across the platform.

So the same style of images used in Find Tasks listings should also be used here.

3. Currency Must Be in Nigerian Naira

All prices on the platform should be displayed in Nigerian Naira (₦).

Currently some prices are shown in dollars, which should not happen.

Please update the entire application so that all monetary values are displayed in Naira.

Examples:

₦5,000

₦10,500

₦2,000

This change should apply to:

Task listings

Payment displays

Task rewards

Anywhere money appears in the system.

4. Category Dropdown Must Be Scrollable

On the Find Tasks page, users can filter tasks using the Category dropdown.

When the list of categories becomes long, users must be able to scroll within the dropdown menu.

This prevents the dropdown from extending beyond the visible screen.

Important details:

The All Categories dropdown must be scrollable.

Users should be able to scroll inside the dropdown to view all categories.

This prevents users from needing to resize their screen to see all options.

Note:

The “All Types” dropdown does NOT need modification.

Only the Category dropdown needs to be scrollable.
