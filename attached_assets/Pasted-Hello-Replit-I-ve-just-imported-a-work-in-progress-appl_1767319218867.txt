1. WHAT HELPCHAIN IS (HIGH-LEVEL)

HelpChain is a real-world, location-based help and service marketplace.

It allows:

People who need help with everyday tasks to post requests

People who can help to discover nearby requests and offer assistance

Safe coordination, communication, and accountability between both sides

This is not:

A charity platform

A donation app

A freelancing marketplace like Fiverr

A professional-only gig site

HelpChain focuses on everyday, practical help, especially in regions where informal work and community-based assistance are common.

Examples of help requests:

Moving items

Cleaning

Errands

Basic repairs

Short local tasks

Assistance for elders

One-time or recurring help

The platform is designed to feel human, local, and trustworthy, not corporate or cold.

2. CORE PHILOSOPHY (THIS GUIDES DESIGN & UX)

Everything in HelpChain must follow these principles:

Simplicity over complexity

Clarity over cleverness

Trust over speed

Local relevance over global abstraction

Professional look without intimidating users

If a feature:

Adds confusion → remove or redesign it

Looks impressive but hurts usability → reject it

Makes users think too hard → simplify

This is not a power-user tool.
This is a real-life people tool.

3. TARGET USERS (VERY IMPORTANT)

HelpChain is for:

Everyday people

Students

Freelancers

Informal workers

Skilled or semi-skilled helpers

Busy individuals who need quick help

People in emerging markets

People who are NOT tech-savvy

Design choices must assume:

First-time internet users

Low patience for confusing layouts

Mobile-first behavior

Trust concerns

Small transaction sizes

4. ACCESS RULES & USER FLOW (CRITICAL)
Logged-Out Users CAN:

View the homepage

Browse help requests

View request summaries

Read platform information

Learn how HelpChain works

Logged-Out Users CANNOT:

View full profiles

Offer help

Post help requests

Message anyone

See sensitive details

If a logged-out user clicks:

“Offer Help”

“View Profile”

“Message”

“Post Request”

They must be prompted to log in or register, clearly and politely.

5. PROFILES (PUBLIC VS PRIVATE)

Every user has one account, but two views:

Public Profile (Visible to others):

Name or display name

Profile photo

Verification status (if applicable)

Number of helps offered

Number of requests posted

Reviews & ratings

Short bio

General location (not exact address)

Private Profile (Owner only):

Wallet balance

Transaction history

Payout details

Account settings

Security info

Absolutely exclude:

Balance

Earnings

Transactions
from public view.

This is non-negotiable.

6. OFFER HELP FLOW (VERY IMPORTANT UX)

When a logged-in user clicks “Offer Help” on a request:

They should see a clean, focused modal or page with:

Summary of the request

Requester name + rating

Location (approximate)

Budget

Time expectations

Input field for:

Message

Availability

Optional proposed price (if flexible)

No clutter.
No unnecessary fields.
No confusion.
